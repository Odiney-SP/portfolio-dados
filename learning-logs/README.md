# Learning Logs

Registro cronológico da jornada de aprendizado — o que foi estudado, o que funcionou e o que ficou para depois.

---

## Como usar

Criar um arquivo por mês: `2026-05.md`, `2026-06.md`, etc.

Cada entrada registra:
- O que foi estudado
- Recursos utilizados
- Principais aprendizados
- Dúvidas pendentes
- Próximos passos

---

## Exemplo de entrada

```markdown
## 2026-05-27

**Estudado:** Window Functions em SQL — ROW_NUMBER, RANK, LAG

**Recurso:** Documentação PostgreSQL + exercícios no DBeaver

**Aprendizado:** A diferença entre RANK() e DENSE_RANK() no tratamento
de empates. LAG() é muito útil para comparar com o período anterior sem JOIN.

**Dúvida pendente:** Quando usar ROWS UNBOUNDED PRECEDING vs RANGE?

**Próximo:** Praticar com dataset de produtividade real
```

---

> Consistência é mais valiosa que intensidade — um log por semana já é suficiente.
